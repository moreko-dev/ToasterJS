Toaster.js

Lightweight javascript notification library

View on Github Quick start

Toaster.js is a lightweight and easy-to-use JavaScript notification library powered by Moreko.

It provides different animations, positions and types to show interesting toasts on the web page.

  • Lightweight and easy-to-use
  • Everything based on object configurations
  • Supports normal, pending and action toasts (New)
  • Customizable in animation, position, duration, direction, ...
  • Supports Limitation and queue system

Ready to start? Visit Toaster.js Github repo here or view a demo of Toaster.js below.

Installation

First get the Toaster.js files from Github repo or download the main files from here.

Then put the files in your project and import the Toaster.min.js and ToastConsts.min.js to your .js file. 

import ToasterJS from "./path/to/files/Toaster.min.js";
import ToastConsts from "./path/to/files/ToastConsts.min.js";

When you import the necessary files to project, You can use the ToasterJS class to get started.

const toaster = new ToasterJS();

Configurations

Toaster.js allows you to config your toasts in many options! Check this out:

const toaster = new ToasterJS({
    animation: ToastConsts.Animation.FLIP,
    direction: ToastConsts.Direction.LTR,
    position: ToastConsts.Position.BOTTOM_CENTER,
    duration: 5000,
    roundedValue: "0.5rem",
    limit: 10,
    queue: true
});

Toasting configs

You can use some options while you are toasting a message, to make it diffrent and usable! Check this out:

const toaster = new ToasterJS( /*options*/ );
toaster.toast("This is a diffrent message!", {
    type: ToastConsts.Type.INFO,
    duration: 3000,
    animation: ToastConsts.Animation.SLIDE_UP,
    roundedValue: "0.25rem"
}, "Title");

In the code above, you can pass an object with optional values of type, duration, animation, roundedValue and an optional title to show a titled toast! (If your title set to Null, the toast is normal!)

Pending toasts

On new update, a pending toast remains visible while a time-consuming operation (e.g. async task) is running. It automatically resolves when the action finishes. You can use pending toast by toastPending().

Check the code below:

toaster.toastPending("Fetching data...", {
        action: () => fetch("/api/data"),
        onSuccess: () => toaster.toast("Data fetched!", { type: ToastConsts.Type.SUCCESS }),
        onError: () => toaster.toast("Failed to fetch data!", { type: ToastConsts.Type.ERROR })
    }, "Loading", {
        animation: ToastConsts.Animation.BOUNCE,
        roundedValue: "0.5rem"
});

In the code above, you should pass 3 important functions:

  • action: Action should be a function that returns a promise and will execute!
  • onSuccess: What to do if the action resolves?
  • onError: What to do if the action reject or catched?

toastPending() gets a title and options object too that on options object, only animation and roundedValue is available.

Action toast

On new update, an action toast shows one or more buttons to trigger specific callbacks.

Check the code below:

toaster.toastAction("Want to close without saving?", [
        { name: "Save", action: () => saveProject() },
        { name: "Don't Save", action: () => closeTab() },
        { name: "Cancel", action: () => {}, closeOnClick: false }
    ], "Confirm", {
        type: ToastConsts.Type.WARNING,
        animation: ToastConsts.Animation.SLIDE_LEFT
});

In the code above, you should pass an array of actions! Each action in the array, is an object with name, action and optional prop closeOnClick. name is the button text too! action is the function that will execute if the related button clicked! closeOnClick is an optional value (Default true) that specifies after the click of button, toast should close or not!
